Introduction

Array: contiguous collection of elements of the same type. Fast random access by index.
String: sequence of characters — often implemented as a character array.

Array — Basics & Memory Layout

Indexing: Most languages use 0-based indexing. Access is O(1) by index for arrays.

Python example (array as list)

arr = [10, 20, 30, 40, 50]
print(arr[0])     # 10
print(arr[4])     # 50
print(arr[-1])    # 50 (Python supports negative indexing)

Memory layout (visual)

Types of Arrays

Common forms: 1D, 2D (matrix), and multi-dimensional arrays. Also static vs dynamic arrays.

Python examples

# 1D
one_d = [1, 2, 3, 4]

# 2D
matrix = [
  [1, 2, 3],
  [4, 5, 6]
]

# indexing: matrix[1][2] -> 6

Common Array Operations

  • Traverse — O(n)
  • Insert at end — amortized O(1) for dynamic arrays
  • Insert/delete mid — O(n)
  • Search — linear O(n), binary O(log n) if sorted
  • Sort — O(n log n) typical

Python demo

arr = [3, 1, 4]
arr.append(5)          # [3,1,4,5]
arr.insert(1, 10)      # [3,10,1,4,5]
arr.remove(1)          # [3,10,4,5]
arr.pop()              # [3,10,4]
arr.sort()             # [3,4,10]
print(arr)
# Output: [3, 4, 10]

Array Algorithms (search & sort)

Linear vs Binary Search

# Linear search (O(n))
def linear_search(arr, target):
    for i, val in enumerate(arr):
        if val == target:
            return i
    return -1

# Binary search (O(log n)) - requires sorted array
def binary_search(arr, target):
    lo, hi = 0, len(arr)-1
    while lo <= hi:
        mid = (lo+hi)//2
        if arr[mid] == target:
            return mid
        if arr[mid] < target:
            lo = mid+1
        else:
            hi = mid-1
    return -1

Sorting

# Python built-in sort (Timsort) - O(n log n) average
arr = [5,2,9,1]
arr.sort()
print(arr)   # [1,2,5,9]

String — Basics & Representation

Strings are sequences of characters. Many languages treat them like arrays of characters, but behavior (mutable vs immutable) varies.

Types of Strings: 1.Mutable(can be changed) and 2.Immutable(once created cannot be changed)

Indexing & Slicing (Python)

s = "Varshu"
print(s[0])      # 'V'
print(s[-1])     # 'u'
print(s[1:4])    # 'ars'
# Output:
# V
# u
# ars

Diagram: string indices & slicing

String Operations — Methods & Examples

Common methods

s = " Hello  Python  "
print(s.strip())           # "Hello  Python"
print(s.upper())           # " HELLO  PYTHON  "
print("Py" in s)           # True
print(s.replace("Python", "World"))

Split / Join / Format

line = "a,b,c"
parts = line.split(",")     # ['a','b','c']
joined = "-".join(parts)    # "a-b-c"
g = f"Hello, {'Varshu'}!"    # "Hello, Varshu!"

CSV parsing example

row = "Alice,21,CS"
name, age, dept = row.split(",")
print(name, age, dept)
# Output:
# Alice 21 CS

Advanced String Topics

Unicode & Encoding

UTF-8 is common. Some characters use multiple bytes — length in characters differs from byte size.

s = "नमस्ते"
print(len(s))    # number of code points -> 6
# But bytes: len(s.encode('utf-8')) > 6

Performance tips

  • Avoid repeated concatenation in loops — build a list and join.
  • Use language-specific mutable builders for heavy string edits (e.g., StringBuilder in Java).

Regex mention

For complex pattern matching, use regular expressions (Python's re module).

Common Pitfalls & Best Practices

  • Index out of range — always validate indices.
  • Confusing dynamic list behavior with static arrays (resizing costs).
  • String immutability — avoid repeated concat in loops.
  • Be mindful of Unicode when slicing bytes vs characters.

Index error example

arr = [1,2,3]
# arr[3] -> IndexError: list index out of range

Summary:

  • Arrays: fast random access, contiguous memory, costly mid-array insertion.
  • Strings: sequences of characters; immutability affects performance choices.
